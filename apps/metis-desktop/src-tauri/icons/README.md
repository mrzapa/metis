# Tauri icon suite

All files in this directory are **generated** from
`apps/metis-web/public/brand/metis-mark.svg` via:

```sh
node apps/metis-web/scripts/build-tauri-icons.mjs
```

Re-run that script whenever the brand SVG changes. Don't hand-edit
the icons directly — they'll be overwritten.

## What's actually bundled

`tauri.conf.json` `bundle.icon` references a deliberate **subset** of
the suite:

```json
"icon": [
  "icons/32x32.png",
  "icons/128x128.png",
  "icons/128x128@2x.png",
  "icons/icon.icns",
  "icons/icon.ico"
]
```

This is Tauri 2's default desktop set:

| File | Platform | Used by |
|---|---|---|
| `32x32.png` | Linux / Windows | Tray icons, small toolbar icons |
| `128x128.png` | Linux / generic | Standard window icon |
| `128x128@2x.png` | macOS / Retina | HiDPI window icon |
| `icon.icns` | macOS | `.app` bundle's `Contents/Resources/` |
| `icon.ico` | Windows | `.exe` resource icon |

The remaining files are generated but **not** bundled by default.
They're available if/when the relevant platform ships:

| Files | When they apply |
|---|---|
| `Square*Logo.png`, `StoreLogo.png` | Microsoft Store packaging (we don't ship there today) |
| `ios/AppIcon-*.png` | Tauri Mobile iOS build (M19 stretch goal — not shipped) |
| `android/mipmap-*/ic_launcher*.png` | Tauri Mobile Android build (M19 stretch — not shipped) |
| `64x64.png`, `icon.png` | Generated by `tauri icon` for completeness; not in the default bundle list |

If we ever start shipping to the Microsoft Store or Tauri Mobile,
expand `bundle.icon` accordingly. Until then, the smaller list keeps
installer size down and avoids confusing platform packagers with
files they don't recognise.

## Source of truth

The 1024×1024 source PNG is regenerated on every script run
(`_source-1024.png`, deleted at the end). The actual
source-of-truth is `apps/metis-web/public/brand/metis-mark.svg` —
treat that as the canonical asset.
