## Intake workflow — new feature requests from external sources

**When the user asks you to implement something that originated outside
this repo** — a GitHub repo, a paper, a tweet, a technique, a concept, a
screenshot, a "wouldn't it be cool if…" — do **not** start coding.

Follow this sequence:

1. **File it to `plans/IDEAS.md` first.** Append a new entry under *Open
   ideas* with: the source (link if given), a one-line summary of what
   the user wants, and any context they provided.
2. **Triage inline.** In the same turn, produce a short review:
   - What it is.
   - Which vision pillar it serves (Cosmos / Companion / Cortex /
     Cross-cutting) or that it doesn't fit the vision.
   - Which existing milestone in `plans/IMPLEMENTATION.md` it overlaps
     with or depends on.
   - A recommendation: **Promote** (worth a milestone row), **Merge**
     (fold into an existing plan), **Park** (Iced — revisit later),
     or **Reject** (with reason).
   - Rough scope: 1-hour patch / multi-day / multi-week.
3. **Ask for a go/no-go.** Do not implement until the user confirms the
   recommendation.
4. **If promoted:** add a row to `plans/IMPLEMENTATION.md` and create a
   plan doc stub under `plans/<slug>/plan.md` using the standard
   frontmatter (Milestone / Status / Claim / Last updated / Vision
   pillar + Progress / Next up / Blockers / Notes). Then and only then
   start implementation against that plan.
5. **If merged:** append to the target plan doc's *Notes for the next
   agent* section. Implement within that existing milestone's scope.
6. **If parked or rejected:** record the reason in `plans/IDEAS.md`
   (move to *Iced* or *Rejected*). Stop.

**Exceptions — the intake step may be skipped when:**
- The user explicitly says "skip triage", "just do it", or equivalent.
- The request is a bug fix, typo, or trivial (<30 min) tweak to
  existing code.
- The work is already claimed by an active `plans/IMPLEMENTATION.md`
  row and the user is asking you to continue it.

Full rules: [`plans/README.md`](../plans/README.md) (*Intake workflow*
section).
